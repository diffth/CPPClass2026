# C/C++ 및 언리얼 엔진 기초 학습 노트
<br>

## 1. 변수 (Variable)
### ① 기본 자료형 (Type)
#### •	숫자
>   정수형: int, short, long 등 (예: 1, 2, 100)
>   실수형: float, double (부동소수점 사용, 예: 1.123)

#### •	문자: char (단일 문자) → string (문자열 클래스)
#### •	논리형: bool (true/false, 1/0)
#### •	상수 (const): 변하지 않는 값. 선언과 동시에 초기화 필수, 이후 값 변경 불가.

### ② 변수 분류
#### •	지역변수 (local): 함수 내부 선언. 함수 호출 시 생성, 종료 시 소멸.
#### •	전역변수 (global): 프로그램 전체 접근 가능. 프로그램 시작 시 생성, 종료 시 소멸.
#### •	정적변수 (static):
>	외부 정적변수: 파일 범위 선언, 다른 파일에서 접근 불가.
>	내부 정적변수: 함수 내 선언, 함수가 종료되어도 값 유지(소멸X).

### ③ 사용자 정의 자료형
#### •	열거형 (enum): 상수들의 집합. (예: enum Color { Red, Green, Blue };)
#### •	구조체 (struct): 여러 타입의 변수 묶음.
#### •	typedef: 기존 자료형에 새로운 별칭 부여.

### ④ 배열 (Array) []
•	같은 타입 변수들의 집합, 연속된 메모리 공간 점유.
•	int arr[5] = {1, 2, 3, 4, 5};
•	1차원, 2차원, 다차원 배열 가능.

### ⑤ std::string 클래스
#### •	문자열을 다루기 위한 클래스 (#include <string>).
> str.length(), str.size(): 길이 반환.
> str.append(), str.insert(), str.replace(): 문자열 조작.
    
### ⑥ 포인터 (Pointer)와 참조자 (Reference)
#### •	포인터 (*): 주소값을 저장하는 변수. &(주소 연산자), *(참조 연산자) 사용.
#### •	참조자 (&): 변수의 별칭. 선언 시 초기화 필수, 대상 변경 불가.
#### •	nullptr: 포인터 초기화 시 사용되는 안전한 널 값.

### ⑦ 동적 메모리 할당
#### •	new / delete: 힙(Heap) 영역에 메모리 할당 및 해제. (Memory Leak 주의)
#### •	스마트 포인터: unique_ptr, shared_ptr 등 자동 메모리 관리 제공.
<br>
    

## 2. 연산자 (Operator)
### •	산술: +, -, *, /, %
### •	대입: =, +=, -=, *=, /=
### •	비교: ==, !=, >, <, >=, <=
### •	논리: && (AND), || (OR), ! (NOT)
### •	etc:
>	삼항 연산자: 조건 ? 참 : 거짓
>	범위 지정 연산자: :: (예: std::cout)
>	sizeof: 메모리 크기 반환.

<br>
    

## 3. 제어문 (Control Flow)
### •	조건문: if, else if, else, switch-case
### •	반복문:
>    while: 조건이 참인 동안 반복.
>    do-while: 최소 1회 실행 후 조건 검사.
>    for: (초기식; 조건식; 증감식) 기반 반복.
>    흐름 제어: break(탈출), continue(다음 루프로 점프).

<br>

## 4. 함수 (Function)
### •	특징: 코드 재사용, 모듈화.
### •	매개변수 전달 방식:
	1. 값 전달 (Call by Value): 값 복사.
	2. 주소 전달 (Call by Address): 포인터 이용.
	3. 참조 전달 (Call by Reference): 참조자 이용.
### •	오버로딩 (Overloading): 이름은 같지만 매개변수가 다른 함수 중복 정의.
### •	템플릿 (Template): 다양한 타입을 지원하는 일반화된 함수/클래스.

<br>
    
## 5. 객체지향 프로그래밍 (OOP)
### ① 핵심 4요소
	1. 추상화: 복잡한 현실을 단순화.
	2. 상속: 부모의 특성을 자식이 물려받음.
	3. 다형성: 하나의 인터페이스로 다양한 구현.
	4. 캡슐화: 데이터 보호 및 정보 은닉.

### ② 클래스와 인스턴스
#### •	클래스: 설계도 (붕어빵 틀).
#### •	객체/인스턴스: 실체 (붕어빵).
#### •	생성자/소멸자: 객체 생성/소멸 시 자동 호출.

### ③ 접근 제한자
#### •	public: 어디서든 접근 가능.
#### •	protected: 자식 클래스까지 접근 가능.
#### •	private: 클래스 내부에서만 접근 가능.

### ④ 다형성과 가상 함수
#### •	가상 함수 (virtual): 자식에서 재정의(Overriding) 가능하도록 선언.
#### •	순수 가상 함수: virtual void Func() = 0; → 자식에서 반드시 구현해야 함.
#### •	추상 클래스: 순수 가상 함수를 포함하여 객체 생성이 불가능한 클래스.

<br>

## 6. 자료구조 및 STL
### ① 선형 구조
#### •	Vector: 동적 배열, 인덱스 접근 빠름.
#### •	List: 연결 리스트, 삽입/삭제 용이.
#### •	Stack: LIFO (후입선출).
#### •	Queue: FIFO (선입선출).

### ② 비선형 구조
#### •	Tree (Set, Map): 부모-자식 관계. Map은 Key-Value 쌍 저장.
#### •	Graph: 정점과 간선의 연결.

<br>

## 7. 빌드, 주석
### •	빌드 과정: 전처리 → 컴파일 → 링킹 → 실행파일(.exe) 생성.
### •	주석:
> //: 한 줄 주석 (Ctrl + /)
> /* ... */: 여러 줄 주석 (Ctrl + Shift + /)
